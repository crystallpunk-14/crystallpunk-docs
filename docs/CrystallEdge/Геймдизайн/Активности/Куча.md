# Активность

Активность - это геймплейный процесс...

- Активность можно выполнить, чтобы получить награды.

- Активности могут быть одноразовыми или многоразовыми. Полезно наверное следить, чтобы у роли не были только одноразовые активности.

- Активности могут быть навязанными, и наказывать за их невыполнение, вместо награды за выполнение (заделка дыр в станции инженерами)

- Ни одна активность не должна быть критически необходимой для работы другой активности но может способствовать ей.

- Активность имеет смысл на свою страничку в гайдбуке

- Дизайн активности состоит из взаимодействия с множеством систем.

# Система

Конкретная C# система, интерфейсы, механика в игровом пространстве.

К дизайну систем подходит оффовский документ по столбам дизайна:

- Система основана на глубоком уровне симуляции процессов.
- Система взаимодействует с одной или больше других систем.
- Система интуитивно понятна.
- Система не требует внешних источников информации для изучения.
- У системы нет очевидного выйгрышного варианта взаимодействия.
- Взаимодействуя с системой игрок может облажаться, веселым или ужасным способом.
- При любых взаимодействиях с системой всегда есть результат, влияющий на дальнейшую игру.
- Система поощряет взаимодействия между разными игроками.
